Permutations (Arrangements) — Core Ideas

A permutation (also called an arrangement) counts the number of ways to place items in order. If the order matters, you are dealing with permutations.

The most common tools are factorials and nPr. Factorial: n! means n × (n−1) × (n−2) × ... × 2 × 1.

Type When to use Method
Full permutation Arrange all n different items in a line n!
Pick and arrange Select r items from n and arrange them (order matters) nPr = n × (n−1) × ... × (n−r+1)
With restrictions Rules like must be together, cannot be together, fixed position Use block method and/or total − invalid
Worked examples

Example 1 (simple): 6 different books are arranged in a row.
Number of arrangements = 6! = 720.

Example 2 (two-step): From 8 students, choose a captain, vice-captain, and secretary.
Number of choices = 8 × 7 × 6 = 336 (this is 8P3).

Example 3 (multi-step restriction): 7 people line up. A and B cannot stand next to each other.
Total = 7!. Invalid (A and B together) = 6! × 2. Valid = 7! − 6! × 2.
